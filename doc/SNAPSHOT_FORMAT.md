# Snapshot Format

This document describes the deterministic artifact structure generated by **codebase-snapshot**.

---

# Core Structural Artifacts

## PROJECT_TREE.txt

Deterministic file tree of the scanned repository.

Purpose:

- structural overview
- repository traversal snapshot
- baseline artifact for downstream analysis

---

## INDEX.tsv

Deterministic repository file index.

Format:

FILE <file> <size> <lines>

Purpose:

- normalized file inventory
- deterministic size and line metadata per file
- base input for higher-level analysis layers including snapshot diffing

---

## DEPENDENCIES.tsv

Deterministic dependency signals extracted from repository files.

Format:

DEPENDENCY <file> <line> <type> <target>

Purpose:

- source/import/include dependency discovery
- repository coupling hints
- dependency-aware interpretation

---

## GRAPH.tsv

Deterministic structural graph of the repository.

Format:

GRAPH <source> <relation> <target>

Typical relations include:

- directory containment
- file roles
- dependency relations
- generated snapshot artifacts

This file helps AI systems quickly understand repository structure and relationships.

---

## SEMANTICS.tsv

Deterministic file-to-component semantic mapping.

Format:

SEMANTIC <file> <component> <group> <role> <rule>

Purpose:

Maps repository files to:

- a high-level component
- a deterministic functional group
- a file role
- an explicit detection rule

This artifact supports component-aware repository interpretation.

---

## ENTRYPOINTS.tsv

Detected execution entrypoints.

Format:

ENTRYPOINT <file> <type> <rule>

Purpose:

Identifies where execution begins within the repository.

---

## MODULES.tsv

Deterministic repository module mapping.

Format:

MODULE <module> <path> <scope> <role> <description> <rule>

Purpose:

Defines the primary architectural zones of the repository.

This artifact is the **first topology layer** above file semantics.

---

## SUBSYSTEMS.tsv

Deterministic repository subsystem mapping.

Format:

SUBSYSTEM <module> <subsystem> <path> <scope> <role> <description> <rule>

Purpose:

Defines stable functional subsystems inside repository modules.

This artifact is the **second topology layer** above file semantics.

---

# Interpretation Artifacts

## PURPOSE.md

Deterministic repository purpose summary.

Purpose:

Explains what the repository is for using deterministic structural signals.

---

## SYSTEM_FLOW.md

Deterministic repository system flow explanation.

Purpose:

Explains **how the repository operates internally**.

The artifact describes:

- the main repository execution entrypoint
- the internal repository analysis pipeline
- interactions between modeling and rendering modules
- how snapshot artifacts are generated

This layer helps humans and AI systems understand repository behavior
without reading the full source code.

---

## REPOSITORY_DNA.md

Deterministic compact repository identity summary.

Purpose:

Provides a compact deterministic identity layer for the repository.

The artifact summarizes:

- project type
- architecture style
- languages
- dependency model
- documentation presence
- test coverage
- repository size
- core components
- entrypoints

Project type detection is hybrid and deterministic:

- structural directory signals first
- entrypoint signals as support

This layer complements PURPOSE.md and REPOSITORY_EXPLAIN.md.

---

## REPOSITORY_EXPLAIN.md

Deterministic high-level repository explanation.

Purpose:

Synthesizes the main repository interpretation layers into one stable
high-level document for both humans and AI systems.

The artifact explains:

- what the repository is
- what it is for
- how it works internally
- where execution begins
- how modules and subsystems are organized
- what snapshot artifacts are generated

This layer acts as the final repository explanation artifact.

---

# Evolution Artifacts

## DIFF.tsv

Deterministic machine-readable snapshot diff.

Format:

DIFF <type> <path>

Types:

- ADDED
- REMOVED
- MODIFIED

Purpose:

- deterministic comparison between two snapshots
- machine-readable repository evolution summary
- base input for human-readable diff rendering

---

## SNAPSHOT_DIFF.md

Deterministic human-readable snapshot evolution summary.

Purpose:

Explains repository evolution between two snapshots using deterministic diff signals.

The artifact summarizes:

- files added
- files removed
- files modified
- high-level interpretation of repository evolution

---

## SNAPSHOT_HISTORY.tsv

Deterministic repository snapshot history index.

Format:

TIMESTAMP <snapshot_id> <snapshot_path>

Purpose:

- persistent repository history index
- deterministic archive listing
- foundation for repository timeline rendering

---

## REPOSITORY_TIMELINE.md

Deterministic human-readable repository timeline.

Purpose:

Renders archived snapshot history as a stable chronological repository view.

The artifact summarizes:

- archived snapshot order
- snapshot timestamps
- snapshot IDs
- generated dates
- tool versions
- labels
- normalized labels
- file counts
- archive paths

This artifact is derived from `SNAPSHOT_HISTORY.tsv` and archived `SNAPSHOT_META.json` files.

---


# Storage Layout

## Main snapshot output

Snapshots generated by the main CLI command are written under:

snapshots/YYYY-MM-DD/vX.Y.Z/<sequence>_<label>/

These remain the primary generated snapshot outputs.

---

## Archived history storage

Archived historical snapshots are stored under:

.snapshots/<snapshot_id>/

The snapshot ID is generated deterministically from UTC time using the format:

snapshot_YYYYMMDD_HHMMSS

These archives are tracked by `SNAPSHOT_HISTORY.tsv`.
