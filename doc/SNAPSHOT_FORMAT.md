# Snapshot Format

A snapshot generated by **codebase-snapshot** produces a deterministic bundle
designed for AI interpretation and repository understanding.

Snapshots are organized using the following structure:

snapshots/YYYY-MM-DD/vX.Y.Z/<sequence>_<label>/

Example:

snapshots/2026-03-09/v0.2.2/01_phase_4_purpose/

Each snapshot directory contains a set of structured artifacts describing the repository.

---

# Core Artifacts

## PROJECT_TREE.txt

Complete list of repository files after exclusion rules.

Purpose:

Provides a deterministic view of the repository structure.

---

## INDEX.tsv

Tab-separated file index.

Format:

FILE <path> <size_bytes> <line_count>

Purpose:

Allows quick statistical and structural analysis of the repository.

---

## DEPENDENCIES.tsv

Extracted dependency signals from source code.

Format:

DEPENDENCY <file> <line> <type> <target>

Examples:

source
import
require
include

Purpose:

Detects syntactic dependency relationships between files.

---

## GRAPH.tsv

Structural relationship graph of the repository.

Format:

GRAPH <source> <relation> <target>

Examples:

GRAPH bin contains bin/snapshot
GRAPH bin/snapshot role cli_entrypoint
GRAPH bin/snapshot depends_on core/naming.sh
GRAPH bin/snapshot generates GRAPH.tsv

Typical relations include:

contains
depends_on
role
generates

Purpose:

Provides a high-level relationship map of the repository:

- directory containment
- file roles
- dependency relations
- generated snapshot artifacts

This file helps AI systems quickly understand repository structure and relationships.

---

## SEMANTICS.tsv

Deterministic file-to-component mapping.

Format:

SEMANTIC <file> <component>

Example:

SEMANTIC bin/snapshot cli
SEMANTIC core/indexer.sh core

Purpose:

Maps repository files to high-level components.

---

## ENTRYPOINTS.tsv

Detected execution entrypoints.

Format:

ENTRYPOINT <file> <type>

Examples:

ENTRYPOINT bin/snapshot cli
ENTRYPOINT main.py application

Purpose:

Identifies where execution begins within the repository.

---

## PURPOSE.md

Deterministic repository purpose summary.

Purpose:

Explains what the repository is for using deterministic structural signals.

Typical contents include:

- repository classification
- operational role
- execution profile
- structural signals supporting the interpretation

This artifact is human-readable and AI-ingestible.

No probabilistic inference is used.

# CODEBASE Export

Structured export of repository source files.

Directory:

CODEBASE/

Files:

01_cli.md
02_core.md
03_tests.md
04_docs.md

Purpose:

Allows AI systems to ingest the source code grouped by repository function.

---

# Metadata and Logs

## LOG.txt

Execution log generated during snapshot creation.

Contains operational messages and diagnostic information.

---

## SNAPSHOT_META.json

Structured metadata describing the snapshot.

Typical fields include:

- tool version
- snapshot timestamp
- target repository path
- snapshot label
- artifact generation status

---

# Deterministic Guarantees

Snapshots generated by the tool must follow strict reproducibility rules:

- deterministic file discovery
- stable ordering
- explicit exclusion rules
- normalized naming
- predictable artifact layout

Running the snapshot twice on the same repository state must produce identical structural outputs.
