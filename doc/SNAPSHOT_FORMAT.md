# SNAPSHOT FORMAT

This document defines the structure of a snapshot generated by **codebase-snapshot**.

A snapshot is a **deterministic bundle describing a repository at a specific moment in time**.

---

# Snapshot Path

Snapshots are generated using the following structure:

snapshots/YYYY-MM-DD/vX.Y.Z/NN_label/

Example:

snapshots/2026-03-08/v0.1.0/01_initial/

Where:

YYYY-MM-DD  → snapshot date  
vX.Y.Z      → tool version  
NN          → numeric sequence  
label       → human readable label  

---

# Snapshot Files

Each snapshot directory contains the following artifacts.

---

AI_INGESTION_GUIDE.md

Guide explaining how an AI system should read the snapshot and in which order.

---

MANIFEST.md

High-level description of the project.

Includes:

- project purpose
- important components
- repository structure

---

ARCHITECTURE.md

Overview of the repository architecture and directory organization.

---

PROJECT_TREE.txt

Deterministic representation of the repository structure.

Generated using a sorted file scan.

---

INDEX.tsv

Deterministic file index of the repository.

Example columns:

TYPE    PATH    SIZE    LINES

Used for fast inspection and tooling.

---

DEPENDENCIES.tsv

Dependency signals extracted from source files.

Each line describes a detected dependency relationship.

Example format:

DEPENDENCY    FILE    LINE    TYPE    TARGET

Examples of detected dependency types:

- source
- import
- from_import
- require
- include

This file helps reconstruct **module relationships and architecture graphs**.

---

DOCUMENTATION.md

Summary or extraction of documentation files found in the repository.

Typically includes:

- Markdown documentation files
- titles extracted from documentation

---

LANGUAGES.md

Summary of detected programming languages and file types.

Example:

Shell: 12 files  
Markdown: 8 files  
JSON: 2 files

This helps AI systems understand the technological composition of the repository.

---

CODEBASE/

Directory containing grouped source code extracts organized for AI ingestion.

Typical structure:

CODEBASE/

01_cli.md  
02_core.md  
03_tests.md  
04_docs.md  

Each file contains raw source grouped by repository area.

---

LOG.txt

Snapshot generation log.

Used for debugging and traceability.

---

SNAPSHOT_META.json

Machine-readable metadata describing the snapshot.

Example fields:

snapshot_version  
tool_version  
generated_at  
target_root  
label  
normalized_label  
sequence  
file_count  

---

# Determinism Requirements

Snapshot generation must guarantee reproducibility.

Rules:

- file discovery must be deterministic
- scanning must use `find` + `sort`
- exclusions must be explicit
- output naming must be normalized

Running the snapshot twice on the same repository state must produce the same structure.

---

# Exclusions

Certain directories should be ignored during scanning.

Typical exclusions include:

.git  
node_modules  
build  
dist  
__pycache__  
snapshots  
logs  

These rules may evolve but must remain explicit and documented.

---

# Human and AI Readability

The snapshot must remain readable by both humans and AI systems.

Human reader needs:

- clear structure
- readable file names
- architecture overview

AI system needs:

- deterministic ordering
- normalized text files
- structured metadata

Both goals must be satisfied simultaneously.
